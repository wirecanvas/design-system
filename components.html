COMPONENT CATALOG · v1

컴포넌트 카탈로그

회사 디자인 시스템의 핵심 컴포넌트 9종을 시각 예시와 코드로 정리했습니다. AI로 사내 서비스를 만들 때 이 파일을 함께 첨부하면 일관된 컴포넌트로 화면이 만들어집니다.

AI와 함께 쓰는 방법

Claude.ai에서 새 대화를 열고, 아래 세 파일을 모두 첨부하세요.

① tokens.css    ② typography.css    ③ components.html (이 파일)

그리고 다음 프롬프트를 사용하세요.

첨부한 tokens.css, typography.css, components.html을 사용해서 [원하는 서비스 설명]을 만들어주세요. 규칙: - components.html에 정의된 컴포넌트만 사용 - 각 컴포넌트의 HTML 구조와 클래스 이름을 그대로 사용 (예: <button class="btn btn-primary btn-md">) - 색상은 var(--color-...) 변수만 사용 - 폰트는 .text-h1, .text-body-1 같은 클래스만 사용 - 임의의 hex 색상값(#5578F0 같은)이나 font-size 직접 입력 금지
01 · BUTTON

Button

사용자 액션을 트리거하는 가장 기본적인 컴포넌트입니다. 중요도에 따라 Primary / Outlined / Text를 구분합니다.

VARIANT — 종류별
SIZE — 크기별
COLOR — 색상별 (강조용)
<!-- Variant --> <button class="btn btn-primary btn-md text-btn-medium">Primary</button> <button class="btn btn-outlined btn-md text-btn-medium">Outlined</button> <button class="btn btn-text btn-md text-btn-medium">Text</button> <!-- Size: btn-sm / btn-md / btn-lg --> <!-- Color: btn-primary / btn-success / btn-error --> <!-- Disabled: disabled 속성 추가 -->
사용 가이드
  • 한 화면에 Primary 버튼은 1~2개로 제한하세요. 너무 많으면 강조가 사라집니다.
  • 취소나 보조 액션은 Outlined 또는 Text 버튼을 사용하세요.
  • 삭제 같은 위험한 액션은 Error 색상을 사용하세요.
  • 버튼 크기는 기본 Medium을 사용하고, 작은 영역에만 Small을 사용하세요.
02 · TEXT FIELD

Text Field

텍스트, 숫자, 이메일 등 사용자 입력을 받는 가장 기본적인 입력 필드입니다.

STATE — 상태별
WITH HELPER — 도움말 포함
회사 이메일을 입력하세요
최소 8자 이상이어야 합니다
<!-- 기본 --> <div class="field"> <label class="field-label text-input-label">이메일</label> <input class="field-input text-input-text" type="email" placeholder="example@company.com"> <span class="field-helper text-helper-text">회사 이메일을 입력하세요</span> </div> <!-- 에러 상태 --> <div class="field field-error"> <label class="field-label text-input-label">비밀번호</label> <input class="field-input text-input-text" type="password"> <span class="field-helper text-helper-text">최소 8자 이상이어야 합니다</span> </div> <!-- 비활성: input에 disabled 속성 추가 -->
사용 가이드
  • 레이블은 항상 입력 필드 위에 배치합니다.
  • 에러 메시지는 구체적으로 작성하세요 (예: "이메일 형식이 올바르지 않습니다").
  • helper text는 입력 가이드나 에러 메시지로 활용합니다.
03 · SELECT

Select

미리 정의된 옵션 중 하나를 선택하게 하는 드롭다운 입력입니다. Text Field와 동일한 스타일에 화살표 아이콘이 추가됩니다.

<div class="field"> <label class="field-label text-input-label">부서 선택</label> <select class="field-input select-input text-input-text"> <option>전체</option> <option>영업팀</option> <option>운영팀</option> </select> </div>
사용 가이드
  • 옵션이 5개 이하면 Radio 그룹을 고려하세요 (선택지가 한눈에 보임).
  • 기본 선택값이 필요하면 첫 옵션을 "전체" 또는 "선택해주세요"로 두세요.
04 · CHECKBOX & RADIO

Checkbox & Radio

여러 항목을 동시에 선택할 때는 Checkbox, 하나만 선택할 때는 Radio를 사용합니다.

CHECKBOX
RADIO
<!-- Checkbox --> <label class="check-wrap"> <input type="checkbox"> <span class="checkbox-box"></span> <span class="text-body-1">라벨 텍스트</span> </label> <!-- Radio (같은 그룹은 name 속성 동일하게) --> <label class="check-wrap"> <input type="radio" name="group-name"> <span class="radio-box"></span> <span class="text-body-1">옵션 A</span> </label>
사용 가이드
  • Checkbox: 여러 개 동시 선택 가능 (예: 관심 분야 다중 선택).
  • Radio: 하나만 선택 가능 (예: 성별, 결제 수단).
  • 옵션을 세로로 나열하려면 flex-direction: column으로 감싸세요.
05 · CHIP

Chip

상태, 카테고리, 태그 등을 작게 표시할 때 사용합니다. 색상으로 의미를 구분합니다.

COLOR VARIANTS
Default Primary Success Warning Error Info Secondary
EXAMPLES — 실제 사용
정상 출근 지각 결근 휴가
<span class="chip text-chip">Default</span> <span class="chip chip-primary text-chip">Primary</span> <span class="chip chip-success text-chip">정상 출근</span> <span class="chip chip-warning text-chip">지각</span> <span class="chip chip-error text-chip">결근</span> <span class="chip chip-info text-chip">휴가</span>
사용 가이드
  • 색상은 의미와 일치하게 사용하세요 (Success=좋음, Error=나쁨, Warning=주의).
  • 한 화면에 너무 많은 색상의 Chip을 섞지 마세요. 시각적으로 산만해집니다.
06 · ALERT

Alert

사용자에게 중요한 정보, 성공/에러 메시지를 알릴 때 사용합니다.

ℹ
알림 제목
참고용 정보 메시지입니다.
✓
성공
요청이 정상적으로 처리되었습니다.
⚠
주의
이 작업은 되돌릴 수 없습니다.
✕
오류 발생
서버와의 연결이 끊어졌습니다.
<!-- Variants: alert-info / alert-success / alert-warning / alert-error --> <div class="alert alert-success"> <span class="alert-icon">✓</span> <div class="alert-body"> <div class="alert-title-text text-alert-title">성공</div> <div class="alert-message text-body-2">요청이 정상적으로 처리되었습니다.</div> </div> </div>
사용 가이드
  • Info: 일반 안내, Success: 작업 완료, Warning: 주의 필요, Error: 문제 발생.
  • title은 짧고 명확하게, message는 구체적으로 작성하세요.
07 · CARD

Card

관련된 정보를 그룹화해서 시각적으로 분리할 때 사용합니다. 헤더 / 본문 / 푸터로 구성됩니다.

고객 응대 요약
2026년 5월 20일 기준

오늘 총 23건의 응대가 진행되었고, 평균 응대 시간은 4분 32초입니다.

<div class="card"> <div class="card-header"> <div class="card-title text-h5">카드 제목</div> <div class="card-subtitle text-body-2">부제목</div> </div> <div class="card-body"> <p class="text-body-1">본문 내용...</p> </div> <div class="card-footer"> <button class="btn btn-text btn-sm text-btn-small">취소</button> <button class="btn btn-primary btn-sm text-btn-small">확인</button> </div> </div>
사용 가이드
  • card-header, card-body, card-footer는 필요한 것만 사용해도 됩니다.
  • 버튼은 card-footer 안에서 오른쪽 정렬됩니다.
08 · TABLE

Table

데이터를 행과 열로 정리해서 보여줄 때 사용합니다. 직원 목록, 출결 기록 등에 적합합니다.

이름 부서 상태 출근 시간
김민지 디자인팀 정상 09:02
이준호 개발팀 지각 09:48
박서연 영업팀 휴가 —
최도윤 운영팀 정상 08:55
<div class="table-wrap"> <table class="table"> <thead> <tr> <th class="text-table-header">이름</th> <th class="text-table-header">부서</th> <th class="text-table-header">상태</th> </tr> </thead> <tbody> <tr> <td class="text-body-2">김민지</td> <td class="text-body-2">디자인팀</td> <td><span class="chip chip-success text-chip">정상</span></td> </tr> </tbody> </table> </div>
사용 가이드
  • 상태/태그 컬럼에는 Chip 컴포넌트를 활용하세요.
  • 컬럼이 많을 때는 가로 스크롤이 자동으로 생깁니다 (table-wrap이 처리).
  • 행에 hover 효과가 자동으로 적용됩니다.
